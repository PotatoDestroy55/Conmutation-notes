Routing entre VLAN con router-on-a-stick

Volver al artículo principal

Configuración para el switch que este conectado al router

Para poder crear una inter-VLAN con router-on-a-stick se necesita seguir estos pasos:

  1. Crear y nombrar las VLAN.

    1. Entramos a la configuración global 

          config t

    2. Introducimos a la sección VLAN junto al número que deseamos crear 

          vlan *vlan id*

    3. Le colocamos un nombre a la VLAN, esto para identificarlo nosotros. 

          name *nombre vlan*

  2. Crear la interfaz de administración.
    A continuación, se crea la interfaz de administración en VLAN 99 junto con el default gateway de R1.
    *Cabe resaltar que esta configuración se necesita hacer al switch que este conectado al router, en este caso podría ser el S1.

    1. Entramos a la configuración global 

          config t

    2. Introducimos a la interfaz vlan 99 (este número puede variar, pero por convención se usa el 99) 

          interface vlan 99

    3. Añadimos la ip que va a tener dicha VLAN. 

          ip add 192.168.99.2 255.255.255.0

    4. Prendemos dicho puerto 

          no shutdown

    5. Nos salimos de la interfaz 

          exit

    6. Y colocamos la ip de puerta de enlace predeterminada 

          ip default-gateway 192.168.99.1

  3. Configurar puertos de acceso (se tiene que colocar los puertos necesarios en su respectiva VLAN)

    1. Entramos a la configuración global 

          config t

    2. Introducimos a la interfaz del puerto que deseamos colocarle (este podria ser con rango o una por una) 

          interface fa0/6

    3. Dentro del puerto configuramos el puerto de acceso en la VLAN necesaria, en este caso la 10 

          switchport mode access
    switchport access vlan 10

    4. Prendemos dicho puerto 

          no shutdown

    5. Nos salimos de la interfaz 

          exit
  4. Configurar puertos de enlace troncal.

    1. Entramos a la configuración global 

          config t

    2. Nos colocamos en la interfaz del puerto que deseamos colocarle el troncal (este primero es para el que esta conectado al otro switch) 

          interface fa0/1

    3. Ahora si habilitamos el modo troncal en el puerto, lo activamos y nos salimos 

          switchport mode trunk
    no shutdown
    exit
    4. Este mismo proceso lo hacemos para el puerto que este conectado el router

      Nos introducimos en el puerto que este conectado, activamos el modo de truncamiento, lo prendemos y nos salimos. 

              interface fa0/5
        switchport mode trunk
        no shutdown
        exit

Configuración para los otros switches

Ahora con los otros switches que se tenga, la configuración es muy similar.

Nota: Estas configuraciones siempre se necesita entrar en el modo de configuracion global.

  1. Se crean las VLAN necesarias y la VLAN que funciona como troncal (es común usar la 99) 

            vlan *vlan id*
        name *nombre VLAN*

  2. Se coloca en la VLAN 99 la ip que tendrá dicho puerto 

            interface vlan 99
        ip add 192.168.99.3 255.255.255.0
        no shutdown
        exit

  3. Se introduce la ip de puerta de enlace predeterminada de dicha VLAN 

            ip default-gateway 192.168.99.1

  4. Con la ip's introducidas, ahora se necesita meter a los puertos conectados con su respectiva VLAN (esto puede hacerse con rangos si es posible) 

            interface fa0/18
        switchport mode access
        switchport access vlan 20
        no shut
        exit

  5. Ahora por último se configura el enlace troncal en el puerto(s) que este conectado al switch(es) 

            interface fa0/1
        switchport mode trunk
        no shutdown
        end

Configuración de subinterfaces para el router proporcionado

  1. Se coloca en la interfaz que es conectado al switch con respecto a la VLAN a tratar
    Para realizar esto es necesario agregarle un "." seguido de un numero por ejemplo 10 es decir ".10" 

            interface G0/0/1.*vlan id*
        interface G0/0/1.10

  2. Ahora para cuestiones de practicidad se le debe colocar una descripción, mencionando que ip de enlace predeterminada esta aludiendo dicha subinterfaz. 

            description Default Gateway for VLAN 10

  3. Con esto ahora si colocamos la respuesta al troncal de la VLAN necesaria (en esta caso la 10) 

            encapsulation dot1Q *vlan id*
        encapsulation dot1Q 10

  4. Por último agregamos la ip que debe ser la puerta de enlace predeterminada de la VLAN 

            ip add 192.168.10.1 255.255.255.0
        exit

  5. Este paso lo tiene que repetir con todas las VLAN que se tenga en los switches, incluyendo a l VLAN nativa 99

            interface G0/0/1.99
        description Default Gateway for VLAN 99
        encapsulation dot1Q 99
        ip add 192.168.99.1 255.255.255.0
        exit

  6. Cuando tenga creada todas las subinterfaces ahora solo queda colocarse en el puerto físico, darle una descripción y activarlo. (al activarlo, activará todas las subinterfaces) 

            interface G0/0/1
        description Trunk Link to S1
        no shutdown
        end

Con esto ya tendrá listo el router-on-a-stick configurado, realice pruebas de ping para ver si lo hizo correctamente.
